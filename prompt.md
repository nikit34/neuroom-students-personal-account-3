# Промпт для создания мобильного клиента личного кабинета ученика

Создай мобильное приложение (React Native / Flutter / нативный) для личного кабинета ученика платформы Нейрум.

## Контекст продукта

Нейрум — платформа для учителей математики и русского языка 5-8 классов. Учителя задают ДЗ, ученики сдают фото решений, получают обратную связь. Основное использование — **смартфоны (95% случаев)**.

## Основные требования к функционалу

### 1. Получение и выполнение домашних заданий

**Получение ДЗ:**
- Главная страница с разделами:
  - "Требуют внимания" — задания со статусом "Пересдай" (красная иконка восклицательного знака)
  - "Текущие задания" — новые, в работе, на проверке (серая иконка часов)
- Страница "Прошлые ДЗ" с группировкой по предметам (не по датам)
- Фильтры по предметам в виде кнопок
- Карточки заданий показывают: дату, предмет, статус, описание, иконку скрепки (если есть фото), стрелку для просмотра

**Сдача ДЗ (максимально просто):**
- Одна большая кнопка "Сделай фото решения" с иконкой камеры
- Один клик для загрузки фото (использовать `capture="environment"` для камеры)
- Предпросмотр загруженного фото
- Возможность удалить и загрузить заново

**Предпроверка родителем:**
- После загрузки фото ученик может отправить задание родителю
- Генерируется уникальная ссылка для родителя
- Кнопка "Копировать" для отправки в Telegram/WhatsApp
- Статус "На проверке у родителя"
- Родитель переходит по ссылке и видит страницу одобрения:
  - Просмотр фото решения
  - Кнопки "Одобрить" и "Вернуть на доработку"
  - После одобрения показывается подтверждение с датой/временем
- Система версий: если ученик вносит изменения после одобрения, создается новая версия
- Видно, какая версия была одобрена родителем (защита от конфликтов)
- После одобрения родителем — статус "Одобрено родителем", затем можно отправить учителю

**Статусы заданий:**
- Новое
- В работе
- На проверке у родителя
- Одобрено родителем
- На проверке у учителя
- Проверено

### 2. Обратная связь от учителя

**Главное — обратная связь, а не оценка:**
- Оценка показывается в футере как вторичная информация
- Основной фокус на обратной связи и рекомендациях
- Заголовок: "Что получилось и что улучшить"
- Разбор ошибок:
  - Сравнение: "Как было у тебя" → "Как должно быть"
  - Описание каждой ошибки с указанием места
  - Заголовок "Почему так" вместо "Описание ошибки"
- Рекомендации по улучшению (нумерованный список)
- Общее резюме от учителя

**Призыв к действию:**
- Большой блок с призывом перейти к AI-репетитору
- Градиентный фон, иконка Sparkles
- Кнопка "Перейти к AI-репетитору"
- Объяснение преимуществ работы с AI-репетитором

### 3. AI-репетитор (параллельный путь обучения)

**Важно:** Это параллельный путь, не требующий пересдачи ДЗ учителю. ДЗ закрывается после дедлайна, но обучение продолжается.

**Чат с AI-репетитором:**
- Интерфейс чата для общения с AI
- Предложенные вопросы для начала диалога
- Интеграция с ошибками из обратной связи
- Баннер: "Это отдельный путь обучения. Тренируйся сколько угодно!"

**Квиз от простого к сложному:**
- После работы с репетитором ученик может пройти квиз
- Вопросы разной сложности (легко → средне → сложно)
- Прогресс-бар показывает текущий вопрос
- Индикатор сложности для каждого вопроса
- Объяснение после каждого ответа
- Визуальная обратная связь (правильно/неправильно)

**Система достижений:**
- После успешного прохождения квиза показывается достижение
- Формат: "Тема done" (например, "Приставки 'при-' и 'пре-' done")
- Анимация при получении достижения
- Статистика правильных ответов
- Страница достижений со всеми полученными достижениями
- Разделение на полученные и в процессе

### 4. Журнал оценок для родителя

**Доступ только родителю:**
- Отдельный роут `/parent/grades`
- Переключение режимов: "Режим родителя" / "Режим ученика"
- Ученик НЕ видит оценки в своем интерфейсе

**Визуализация оценок:**
- Карточки по каждому предмету
- Общая статистика вверху:
  - Средний балл по всем предметам
  - Общее количество оценок
  - Количество предметов
  - Разбивка по категориям (Отлично, Хорошо, Удовлетворительно, Неудовлетворительно)
- Средняя оценка и общее количество по предмету
- Индикатор тренда (улучшение/снижение)
- График оценок по датам
- Список оценок с датами и описаниями заданий

### 5. Навигация

**Нижняя навигация (мобильная):**
- Главная
- Прошлые ДЗ
- Достижения
- Оценки (ведет в журнал родителя)

**В режиме родителя:**
- Журнал оценок
- Вернуться (в режим ученика)

### 6. Мобильная оптимизация

**Критически важно:** 95% использования на смартфонах

**Требования:**
- Адаптивный дизайн с breakpoints: 768px (планшеты), 480px (смартфоны)
- Touch-friendly интерфейс:
  - Минимальный размер элементов 44px (стандарт Apple/Google)
  - Убрать hover-эффекты на мобильных
  - Отключить подсветку при тапе
- Оптимизированные размеры шрифтов для чтения на маленьких экранах
- Поддержка safe-area-inset для устройств с вырезами
- Компактная навигация с иконками
- Кнопки: минимальная высота 48px, полная ширина на мобильных
- Viewport с `user-scalable=no` для предотвращения случайного масштабирования

**Оптимизация компонентов:**
- Большие кнопки для загрузки фото
- Удобная прокрутка
- Адаптивные карточки
- Оптимизированные формы

### 7. Дизайн и UX

**Цветовая схема:**
- Фиолетовый (#8B5CF6) — основной цвет
- Розовый (#EC4899) — вторичный цвет
- Светло-фиолетовый фон (#FAF5FF)
- Белые карточки с тенями

**Принципы:**
- Максимально простой интерфейс
- Минимум шагов для выполнения действий
- Понятная визуальная иерархия
- Плавные анимации
- Современный, чистый дизайн

### 8. Метрики успешности

**Ключевые метрики для отслеживания:**
1. Конверсия в AI-репетитор (цель: >40%)
2. Доля выполненных заданий (цель: >70%)
3. Прохождение полного цикла обучения (цель: >25%)
4. Просмотр обратной связи (цель: >80%)
5. Retention (DAU/MAU) (цель: >0.3)

**Критерии оценки "сейчас":**
- Интересный подход (субъективная оценка)
- Техническое исполнение (скорость, UI/UX, вау-эффект)
- Понимание вовлеченности в продукт

## Технические требования

- React Native / Flutter / нативный (iOS + Android)
- TypeScript
- Адаптивная верстка для всех размеров экранов
- Оптимизация производительности для мобильных устройств
- Поддержка камеры для загрузки фото
- Интеграция с системой отправки ссылок (Telegram/WhatsApp)
- Система версий для ДЗ
- Чат с AI-репетитором
- Система квизов и достижений

## Структура приложения

1. **Главная** — обзор заданий (требуют внимания, текущие)
2. **Прошлые ДЗ** — список сданных заданий (группировка по предметам)
3. **Детальная страница задания** — описание, загрузка фото, отправка родителю/учителю
4. **Обратная связь** — разбор ошибок, рекомендации, переход к AI-репетитору
5. **AI-репетитор** — чат, квиз, достижения
6. **Достижения** — все полученные достижения
7. **Журнал оценок (родитель)** — визуализация оценок по предметам
8. **Одобрение родителем** — страница по ссылке для одобрения ДЗ

## Важные моменты

- Оценки скрыты от ученика (только обратная связь)
- Предпроверка родителем — опциональная, но важная функция
- AI-репетитор — параллельный путь, не требует пересдачи ДЗ
- Фокус на обучении и улучшении, а не на оценках
- Максимальная простота использования
- Мобильная оптимизация критически важна

Создай полнофункциональное мобильное приложение с современным дизайном, оптимизированное для смартфонов, с плавными анимациями и отличным UX.
